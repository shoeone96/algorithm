# Grid BFS 유형 정리

> **범위**: Part 6 Grid/암시적 그래프 (W11~W12 풀이 기반)
> **핵심 질문**: "어디서 BFS를 시작할 것인가?"

---

## 유형별 분류

### 1. 연결 요소 탐색

각 미방문 셀마다 BFS를 시작해서 연결된 영역을 탐색.

| 문제 | 핵심 |
|------|------|
| NumberOfIsland | 연결 요소 개수 카운팅 |
| MaxAreaOfIsland | 연결 요소 크기 추적 |
| CountOfCloud | NumberOfIsland 변형 |
| FloodFill | 시작점에서 연결된 영역 채우기 |

**패턴**: 이중 for문 → 미방문 & 조건 맞는 셀 발견 → BFS → 방문 처리

### 2. 단일 출발점 최단거리

특정 한 점에서 출발해서 목표까지 최단 거리.

| 문제 | 핵심 |
|------|------|
| ShortestPathInBinaryMatrix | (0,0) → (n-1,n-1) 최단 경로, 8방향 |

**패턴**: 시작점 큐에 넣기 → BFS 레벨별 탐색 → 도착하면 거리 반환

### 3. 멀티소스 BFS

조건에 맞는 셀을 **전부** 큐에 넣고 동시에 출발.

| 문제 | 핵심 |
|------|------|
| RottenOrange | 썩은 토마토에서 동시 출발 → 전파 시간 |
| MatrixOne (01 Matrix) | 0인 셀에서 동시 출발 → 각 1까지의 거리 |
| AsFarFromLandAsPossible | 육지에서 동시 출발 → 가장 먼 바다까지 거리 |

**패턴**: 조건 맞는 셀 전부 큐에 → BFS → grid 값으로 거리 기록
**사고 전환**: "각 셀에서 가장 가까운 X까지 거리" → "X에서 동시에 출발하면 자동으로 최단 거리"

### 4. 경계 역방향 BFS

경계(가장자리)에서 안쪽으로 BFS. 정방향이 비효율적일 때 뒤집어 생각.

| 문제 | 핵심 |
|------|------|
| SurroundedRegions | 경계 O에서 BFS → 연결된 O는 보존, 나머지 뒤집기 |
| PacificAtlanticWaterFlow | 두 바다 경계에서 각각 역방향 BFS → 교집합 |

**패턴**: 경계 셀 큐에 → 안쪽으로 BFS → "여기에 속하는가" 판별
**사고 전환**: "이 셀이 경계에 도달하는가?" → "경계에서 이 셀에 도달하는가?"

### 5. 8방향 BFS

4방향(상하좌우) 대신 대각선 포함 8방향.

| 문제 | 핵심 |
|------|------|
| ShortestPathInBinaryMatrix | 8방향 최단 경로 |
| Ambulance | 8방향 최단 거리 |

**패턴**: `go` 배열을 8방향으로 확장. 나머지 BFS 로직은 동일.

## 공통 템플릿

```
1. 시작점 결정 (유형에 따라 다름)
2. 큐에 시작점 넣기 + 방문 처리
3. while (!queue.isEmpty())
   - poll
   - 4/8방향 탐색
   - 범위 체크 + 조건 체크 + 미방문 체크
   - 큐에 넣기 + 방문 처리 (넣을 때 처리!)
```

> **주의**: 방문 처리는 큐에 넣을 때 해야 중복 방문 방지. poll할 때 하면 같은 셀이 여러 번 큐에 들어감.

## 유형 판별 흐름

```
문제를 읽는다
  → "최단 거리/시간" 키워드? → BFS
  → 시작점이 여러 개? → 멀티소스 BFS
  → "경계에 닿는가/닿지 않는가"? → 경계 역방향 BFS
  → "연결된 영역 개수/크기"? → 연결 요소 탐색
  → 대각선 이동 허용? → 8방향 BFS
```
