# 01 Matrix 풀이 회고 - 멀티소스 BFS

> **문제**: 01 Matrix (LeetCode 542)
> **유형**: 멀티소스 BFS (Grid)
> **코드**: `MatrixOne.java` (개별 BFS, TLE), `MatrixOne2.java` (멀티소스 BFS)

---

## 1. 핵심 아이디어

각 셀에서 가장 가까운 0까지의 거리를 구하는 문제.

- 모든 0을 **소스(거리 0)**로 큐에 넣고 동시에 BFS
- BFS 레벨 단위 탐색 → **먼저 도달 = 최단 거리 보장**
- isVisited로 중복 방문 차단해도 최단 거리에 영향 없음

## 2. 삽질 과정

### 첫 시도: 1에서 개별 BFS (MatrixOne.java) → TLE
- 값이 1인 각 셀마다 BFS로 가장 가까운 0 탐색
- 시간복잡도: **O(m²n²)** → 시간 초과

### 관점 전환: 0에서 멀티소스 BFS (MatrixOne2.java)
- "각 1마다 BFS를 돌아야 할 것 같다" → **반대편(0)에서 시작하면 한 번에 끝남**
- 시간복잡도: **O(m×n)**

### 버그: continue로 전파 차단
- 1인 셀에서 거리만 기록하고 `continue` → 더 먼 1에게 전파 불가
- **수정**: 1인 셀도 큐에 넣어서 계속 퍼뜨려야 함

### 성능: isVisited를 큐에 넣을 때 처리
- poll할 때 visited 체크 → 같은 셀이 큐에 여러 번 들어감
- **수정**: offer할 때 visited 처리하여 중복 삽입 방지

## 3. 사고 패턴 정리

| 패턴 | 적용 |
|------|------|
| 역방향 멀티소스 BFS | "각 셀마다 BFS 돌아야 할 것 같다" → 반대편에서 시작하면 한 번에 끝나지 않을까? |
| BFS 선점 = 최단 거리 | 레벨 단위 탐색이므로 먼저 도달한 거리가 곧 최단 거리 |
| 소스 판별 | 거리가 0인 쪽이 소스 (Rotten Oranges: 썩은 오렌지, 이 문제: 0인 셀) |
| visited 타이밍 | offer할 때 처리해야 중복 삽입 방지 |
